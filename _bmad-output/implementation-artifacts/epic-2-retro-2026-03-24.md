## 📚 Lessons Learned

### Technical

1. **Date arithmetic trong SQL cần test thực tế với real dates.** `2026-04-07` trông như Monday nhưng thực ra là Thứ Ba. pgTAP tests phát hiện điều này. Lesson: luôn verify ngày bằng `EXTRACT(DOW FROM date)` trước khi dùng làm fixture.

2. **Timezone display bugs chỉ thấy khi render thực tế.** `ScheduleDeadlineBadge` hiển thị UTC thay vì ICT — không detect được bằng unit test hay code review. Cần visual testing với real data.

3. **3-tier lock model tốt hơn timestamp-based lock.** `slot_date < today` (day-based) rõ ràng hơn nhiều so với `now() >= slot.start_time` (UTC exact) từ góc nhìn business logic. Đơn giản để implement và dễ giải thích cho user.

4. **"Reason required for all edits" quá friction.** Chỉ áp dụng reason/notification cho edits trong current week (đã committed) là đúng. Future week = drafting mode, không cần audit trail.

### Process

5. **Story spec deviation cần được surface ngay, không phải khi retro.** Dev agent nên note rõ trong Dev Agent Record khi implementation differs từ AC.

6. **2 loại "done" khác nhau:** sprint-status `done` (story complete) và story file `Status: done` (verified). Cần update cả hai cùng lúc — đây là pattern sẽ enforce từ Epic 3.

7. **Screenshot review by PL là valuable testing layer.** Thắng phát hiện 2 bugs + 4 UX issues + clarified business requirements trong một session. Đây là informal UAT nên institutionalize.
